# PRD — BandSheet 產品需求文件

## 1. 目標使用者與情境
- 業餘/半職業樂團成員(3–8 人)
- 情境:練團前主唱改了歌詞、吉他手改了和弦進行,大家需要看到同一份最新版;練團時用手機/平板看譜。

## 2. 角色與權限
| 角色 | 權限 |
|------|------|
| Owner | 管理樂團、邀請/移除成員、刪除歌曲 |
| Editor | 建立/編輯歌曲譜 |
| Viewer | 只能檢視 |

## 3. 核心功能(v1,必須)

### 3.1 帳號與樂團
- Email + 密碼註冊/登入(JWT)
- 建立樂團(Band),一人可屬於多個樂團
- 以邀請連結(帶 token,7 天有效)邀請成員,加入時指定角色

### 3.2 歌曲與譜
- 歌曲(Song)屬於一個樂團,欄位:歌名、原唱/作者、原調(key)、速度(BPM)、拍號、標籤
- 譜內容以 ChordPro 變體純文字儲存(見 CHORD_SPEC.md)
- 編輯器:左側純文字編輯、右側即時渲染預覽(和弦顯示在歌詞正上方)
- 段落標記:[Verse]、[Chorus]、[Bridge]、[Intro]、[Outro]、[Solo] 等
- 一鍵移調 ±1 半音、或直接選目標調;可切換顯示 Capo 換算
- 歌曲列表:搜尋(歌名)、依標籤篩選、依最近更新排序

### 3.3 即時共編
- 同一首歌可多人同時開啟
- 顯示目前在線的成員(presence,頭像列)
- 編輯以「段落」為單位鎖定:某人正在編輯某段落時,其他人該段落唯讀並顯示「XX 編輯中」
- 段落編輯完成(失焦或 3 秒 idle)即廣播更新,其他人即時看到

### 3.4 版本歷史
- 每次「儲存」建立一個版本(快照全文)
- 版本列表:時間、作者、可選填備註
- 可檢視任一版本、一鍵還原(還原本身也產生新版本)

### 3.5 檢視模式(練團用)
- 全螢幕檢視模式:大字體、可調字級、自動捲動(可調速度、開/關)
- 深色模式

## 4. v2(不要現在做,但架構要能容納)
- 匯出 PDF
- setlist(演出歌單,拖拉排序)
- 音訊參考連結(YouTube/Spotify embed)
- 字元級即時協作(CRDT)

## 5. 非功能需求
- 譜更新廣播延遲 < 1 秒(同區域)
- 支援行動裝置瀏覽器(檢視模式優先做 RWD)
- 單一樂團最多 200 首歌不出現明顯效能問題
- 密碼以 BCrypt 儲存;所有 API 需驗證且做樂團層級授權檢查

## 6. 明確不做(v1)
- OAuth 第三方登入
- 五線譜/tab 譜渲染
- 離線編輯
- 檔案上傳
