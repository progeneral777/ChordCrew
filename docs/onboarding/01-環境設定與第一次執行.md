# 01 · 環境設定與第一次執行

目標:**把整個 BandSheet 在你自己電腦上跑起來**,並在瀏覽器看到畫面。
這一章不需要你懂任何程式,照著打指令就好。

---

## 0. 你會用到的名詞(先有個印象就好)

- **終端機 / Terminal**:打指令的黑視窗。Mac 上叫「終端機」,可在 Spotlight 搜尋。
- **`指令`**:看到程式碼框裡的東西,就是要你複製貼到終端機按 Enter。
- **repo(倉庫)**:整個專案的程式碼資料夾,用 git 管理。

---

## 1. 安裝需要的工具

### 1-1. Git(下載程式碼用)
Mac 通常內建。確認:
```bash
git --version
```
沒有的話:`brew install git`(brew 是 Mac 的套件管理器,見下)。

### 1-2. Homebrew(Mac 套件管理器,裝其他工具用)
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### 1-3. Docker Desktop(一鍵跑整個網站,最推薦)
到 <https://www.docker.com/products/docker-desktop/> 下載安裝,打開它(選單列會出現一隻鯨魚)。
確認:
```bash
docker --version
```

> **只想「先看它跑起來」的人,裝到這裡就夠了** —— 直接跳到第 3 節用 Docker 跑。
> 想「改程式碼、邊改邊看」的人,再繼續裝下面的 Java 和 Node。

### 1-4. Java 21(改後端才需要)
```bash
brew install openjdk@21
```
安裝後要讓終端機「找得到」Java 21。**每次開新終端機、要跑後端前**,先執行:
```bash
export PATH="/opt/homebrew/opt/openjdk@21/bin:$PATH"
```
確認:
```bash
java -version   # 要看到 21.x
```
> 想一勞永逸:把上面那行 `export ...` 加到你的 `~/.zshrc` 檔案最後面。

### 1-5. Node.js(改前端才需要)
```bash
brew install node
```
確認:
```bash
node -v   # 建議 20 以上
npm -v
```

## 2. 取得程式碼

```bash
git clone <這個專案的 git 網址>
cd ChordCrew
```
(如果你已經有這個資料夾,就 `cd` 進去即可。)

---

## 3. 最簡單:用 Docker 一鍵跑整個網站

在專案根目錄(有 `docker-compose.yml` 的地方):
```bash
docker compose up -d --build
```
第一次會比較久(要下載、編譯),請耐心等。跑完後:

- 打開瀏覽器到 **<http://localhost:5173>** → 這就是 BandSheet 的網站!
- 註冊一個帳號 → 建立樂團 → 建立歌曲 → 開始玩。

**它幫你跑了三個東西(叫「容器 container」):**
| 容器 | 是什麼 | 對外的埠(port) |
|---|---|---|
| `chordcrew-db-1` | PostgreSQL 資料庫 | 5432 |
| `chordcrew-backend-1` | Java 後端 | 8090 |
| `chordcrew-frontend-1` | 前端網頁(用 nginx 提供) | 5173 |

常用指令:
```bash
docker compose ps          # 看三個容器活著沒
docker compose logs -f backend   # 看後端的即時日誌(排錯用)
docker compose down        # 全部關掉
docker compose up -d --build     # 改了程式碼後,重新編譯並啟動
```

> 如果 `docker` 指令卡住或報 I/O error,通常是 Docker Desktop 自己出問題:
> 重開 Docker Desktop(選單列鯨魚 → Quit,再重新打開)即可。

---

## 4. 開發模式:邊改程式碼邊看結果(熱重載)

Docker 適合「跑整包」,但你在「改程式碼」時,會想改完馬上看到效果。
這時改用「開發模式」:資料庫用 Docker 跑,前後端在本機直接跑。

**開三個終端機分頁:**

**分頁 A — 只用 Docker 跑資料庫:**
```bash
docker compose up -d db
```

**分頁 B — 後端(改 Java 會用到):**
```bash
export PATH="/opt/homebrew/opt/openjdk@21/bin:$PATH"
cd backend
./gradlew bootRun
```
看到 `Started BandsheetApplication` 就代表後端起來了(在 <http://localhost:8090>)。

**分頁 C — 前端(改 React 會用到):**
```bash
cd frontend
npm install     # 只有第一次要跑,安裝套件
npm run dev
```
打開 <http://localhost:5173>,改任何 `.tsx` 檔案存檔,畫面會**自動更新**。

> 前端怎麼跟後端講話?前端的 `vite.config.ts` 設定了「代理 proxy」:
> 前端收到 `/api/...` 的請求,自動轉發到後端的 `http://localhost:8090`。
> 所以你在瀏覽器只需要開 5173,不用管 8090。

---

## 5. 跑測試(確認你沒改壞東西)

**後端測試(JUnit):**
```bash
export PATH="/opt/homebrew/opt/openjdk@21/bin:$PATH"
cd backend && ./gradlew test
```

**前端測試(Vitest)與檢查:**
```bash
cd frontend
npm run test     # 跑單元測試
npm run lint     # 檢查程式碼風格
npm run build    # 確認 TypeScript 型別沒錯、能打包
```

> 貢獻程式碼前,請確保這些都「綠燈」(pass)。詳見 [05](./05-動手做第一個功能.md)。

---

## 6. 卡住了?

- Java 找不到 → 你忘了 `export PATH=...`(每個新終端機都要)。
- 埠被占用(port already in use)→ 有別的程式占用 5173/8090/5432,先關掉,或 `docker compose down`。
- Docker 卡住 → 重開 Docker Desktop。
- 更多問題看 [06-詞彙表與FAQ](./06-詞彙表與FAQ.md)。
