# 05 · 動手做第一個功能(跨前後端)

讀完前面幾章,這一章帶你**實際動手**做一個小功能,把「資料庫 → 後端 → 前端」整條串起來。
做完你就會有信心接真正的任務了。

---

## Part A. 開發流程(每次貢獻都照這個走)

### 1. 開一個分支(branch)
不要直接在 `main` 上改。先開自己的分支:
```bash
git checkout -b feat/playlist-note      # 分支命名:類型/簡短說明
```

### 2. 改程式碼(見 Part B)

### 3. 跑測試,確認沒改壞
```bash
# 後端
export PATH="/opt/homebrew/opt/openjdk@21/bin:$PATH"
cd backend && ./gradlew test
# 前端
cd frontend && npm run lint && npm run test && npm run build
```
全部綠燈才算過關。

### 4. 提交(commit),用「Conventional Commits」格式
訊息開頭要標類型:
- `feat:` 新功能
- `fix:` 修 bug
- `chore:` 雜項(設定、文件以外的瑣事)
- `docs:` 文件
```bash
git add .
git commit -m "feat: 歌單可加備註"
```

### 5. 推上去、開 Pull Request(PR)
```bash
git push -u origin feat/playlist-note
```
到 GitHub 開 PR,說明「做了什麼、為什麼」。請人 review,通過後合併回 `main`。

> 心法:**小步提交、常跑測試、寫清楚 commit 訊息**。這比一次寫一大包好維護太多。

## Part B. 練習:幫「歌單」加一個「備註」欄位

我們要讓每個歌單可以填一段備註(例如「這場走民謠風,吉他降半音」)。
這個練習會碰到**每一層**,是最好的入門。跟著做:

> 這是練習。做完你可以保留、也可以 `git checkout .` 還原。重點是走一遍流程。

### 步驟 1:資料庫 —— 新增遷移檔
在 `backend/src/main/resources/db/migration/` 建立 `V6__playlist_note.sql`:
```sql
ALTER TABLE playlists ADD COLUMN note VARCHAR(500);
```
> 為什麼是 V6?因為現在最大到 V5,遷移檔要**接續編號**。

### 步驟 2:Entity —— 讓 Java 物件也有這個欄位
編輯 `backend/src/main/java/com/bandsheet/playlist/Playlist.java`,加欄位與 getter/setter:
```java
    @Column(length = 500)
    private String note;

    public String getNote() { return note; }
    public void setNote(String note) { this.note = note; }
```

### 步驟 3:DTO —— 讓 API 回傳/接收這個欄位
編輯 `playlist/dto/PlaylistDtos.java`。在 `PlaylistDetail` 加上 `note`:
```java
    public record PlaylistDetail(UUID id, String name, String note,
                                 List<PlaylistSongItem> songs, Instant updatedAt) {}
```
再加一個更新備註的請求 record:
```java
    public record UpdateNoteRequest(@Size(max = 500) String note) {}
```
> 因為 `PlaylistDetail` 多了參數,**所有 `new PlaylistDetail(...)` 的地方都要補上 note**。
> 編譯器會告訴你哪裡漏了(這正是靜態型別的好處)。在 `PlaylistService.toDetail(...)` 把 `p.getNote()` 帶進去。

### 步驟 4:Service —— 商業邏輯
在 `PlaylistService.java` 加一個方法:
```java
    @Transactional
    public PlaylistDetail updateNote(UUID userId, UUID playlistId, String note) {
        Playlist p = requirePlaylist(userId, playlistId);   // 已存在的權限檢查:限本人
        p.setNote(note);
        return toDetail(p);
    }
```

### 步驟 5:Controller —— 開一條 API
在 `PlaylistController.java` 加:
```java
    @PatchMapping("/{id}/note")
    public ApiResponse<Map<String, PlaylistDetail>> updateNote(
            @AuthenticationPrincipal AuthUser me,
            @PathVariable UUID id,
            @Valid @RequestBody UpdateNoteRequest req) {
        return ApiResponse.ok(Map.of("playlist", playlistService.updateNote(me.id(), id, req.note())));
    }
```
到這裡後端就完成了。重啟後端(`./gradlew bootRun` 或 `docker compose up -d --build backend`),
Flyway 會自動跑 V6。可以用瀏覽器 DevTools 或 curl 測 `PATCH /api/me/playlists/{id}/note`。

### 步驟 6:前端 api —— 包一個函式
編輯 `frontend/src/api/playlists.ts`。在型別 `PlaylistDetail` 加 `note: string | null`,
並加函式:
```ts
  updateNote: (id: string, note: string) =>
    client.patch<{ data: { playlist: PlaylistDetail } }>(`/me/playlists/${id}/note`, { note }),
```

### 步驟 7:前端頁面 —— 顯示與編輯
編輯 `frontend/src/features/playlists/PlaylistDetailPage.tsx`,在歌單名稱底下顯示備註,
並加一個可編輯的欄位(可仿照該檔已有的「改名」互動:用 `useState` 存輸入、按鈕呼叫
`playlistsApi.updateNote(id, note)`,成功後 `apply(res.data.data.playlist)` 更新畫面)。

### 步驟 8:測試、提交
```bash
cd backend && ./gradlew test
cd frontend && npm run lint && npm run test && npm run build
git add . && git commit -m "feat: 歌單可加備註"
```

**恭喜!** 你剛剛完成了一個「資料庫 → Entity → DTO → Service → Controller → api → 頁面」的完整垂直切片。
這就是這個專案 90% 功能的開發方式。

## Part C. 常見坑(新手最容易卡的地方)

| 症狀 | 原因 / 解法 |
|---|---|
| `java: command not found` 或版本不對 | 忘了 `export PATH="/opt/homebrew/opt/openjdk@21/bin:$PATH"`。每個新終端機都要。 |
| 後端起不來,說 schema validation 失敗 | 你改了 Entity 卻沒寫對應的遷移(或反之)。Entity 與資料表結構要一致。 |
| 改了 DTO 後編譯一堆紅字 | record 參數變了,呼叫端要一起改。照編譯器指的位置補齊。 |
| 前端 `res.data.data is undefined` | 後端回傳形狀和你在 api 檔寫的型別對不上,或 API 根本回了錯誤。開 DevTools → Network 看實際回應。 |
| 前端畫面沒更新 | 你直接改了物件而沒用 `setXxx(...)`。React 只有透過 setState 才會重繪。 |
| CORS / 401 一直跳 | token 沒帶到或過期。確認有登入;`client.ts` 會自動帶 token。 |
| Docker 指令卡住 / I/O error | 重開 Docker Desktop。 |
| 埠被占用 | `docker compose down`,或找出占用 5173/8090/5432 的程式關掉。 |
